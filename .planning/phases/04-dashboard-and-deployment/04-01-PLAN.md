  <action>
    1. Create `app/dashboard/__init__.py` — Blueprint("dashboard", __name__), import routes at bottom (same pattern as assessment/__init__.py and documents/__init__.py).

    2. Create `app/services/deadline_service.py`:
       - `from __future__ import annotations` at top (Python 3.14 safety).
       - `ClaimDeadlines` dataclass with fields: limitation_deadline (Optional[date]), defence_deadline (Optional[date]), settlement_conf_date (Optional[date]), trial_request_deadline (Optional[date]), overdue_items (list[str]).
       - `build_claim_deadlines(claim, today: date | None = None) -> ClaimDeadlines` pure function.
       - Reads limitation deadline from `claim.step_data["limitation"]["basic_deadline"]` (ISO string).
       - Reads service_date from `claim.step_data["claim_dates"]["service_date"]` — calculates defence_deadline as service_date + DEFENCE_DEADLINE_DAYS (20).
       - Reads settlement_conference_date from `claim.step_data["claim_dates"]["settlement_conference_date"]` — stores as settlement_conf_date, calculates trial_request_deadline as settlement_conference_date + TRIAL_REQUEST_DEADLINE_DAYS (30).
       - Populates overdue_items list with labels of any deadlines that are past today.
       - Uses `dateutil.relativedelta` for date math (already installed, same as limitation_service.py).
       - All dates that are None (user hasn't entered them yet) are left as None — no errors.
       - Import constants from `app.ontario_constants`: DEFENCE_DEADLINE_DAYS, TRIAL_REQUEST_DEADLINE_DAYS.

    3. Create `app/dashboard/routes.py`:
       - `from __future__ import annotations` at top.
       - Import: dashboard_bp from app.dashboard, db from app.extensions, login_required + current_user from flask_login, selectinload from sqlalchemy.orm, Claim from app.models.claim, Document from app.models.document, build_claim_deadlines from app.services.deadline_service.
       - `GET /` route (dashboard.index): @login_required. Query claims with `db.select(Claim).where(Claim.user_id == current_user.id).options(selectinload(Claim.documents)).order_by(Claim.updated_at.desc())`. Query documents separately with `db.select(Document).where(Document.user_id == current_user.id).order_by(Document.updated_at.desc())`. Build claim_deadlines dict: `{c.id: build_claim_deadlines(c) for c in claims}`. Compute has_overdue flag (any claim has non-empty overdue_items). Pass `claims`, `documents`, `claim_deadlines`, `has_overdue`, and `today=date.today()` to template.
       - `POST /dashboard/dates/<int:claim_id>` route (dashboard.save_dates): @login_required. Load claim with `db.session.get(Claim, claim_id)`. Verify `claim.user_id == current_user.id`, else abort(404). Read `service_date` and `settlement_conference_date` from `request.form`. Validate each: strip, attempt `date.fromisoformat()`, skip invalid. Build new step_data dict: `step_data = dict(claim.step_data or {})`, set `step_data["claim_dates"] = claim_dates`, reassign `claim.step_data = step_data` (CRITICAL: full reassignment, not in-place mutation — MutableDict only tracks top-level changes). `db.session.commit()`. Check `HX-Request` header: if HTMX, return rendered `dashboard/partials/deadline_timeline.html` partial with fresh `build_claim_deadlines(claim)` and `today=date.today()`. Otherwise redirect to `dashboard.index`.

    4. Register dashboard blueprint in `app/__init__.py`: Import `dashboard_bp` from `app.dashboard`, call `app.register_blueprint(dashboard_bp)` alongside existing blueprints (no url_prefix).

    5. Update `app/main/routes.py`: Change the `index()` route to redirect to `dashboard.index` instead of rendering the placeholder template. Import `redirect, url_for` from flask. The route becomes: `return redirect(url_for("dashboard.index"))`. This way `/` redirects logged-in users to the dashboard.

    6. Update `app/templates/main/index.html` to be a simple redirect notice (or delete its content — it won't be rendered anymore since index() redirects).
  </action>
