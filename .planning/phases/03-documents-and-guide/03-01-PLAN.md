**1. Create `app/models/document.py`:**

IMPORTANT: `from __future__ import annotations` at top (Python 3.14 requirement).

- `DocumentType(str, enum.Enum)`: DEMAND_LETTER, FORM_7A, FORM_9A
- `Document(db.Model)` on table `documents`:
  - `id: Mapped[int]` primary key
  - `user_id: Mapped[int]` FK to users.id with CASCADE, indexed
  - `claim_id: Mapped[Optional[int]]` FK to claims.id with SET NULL, nullable (standalone docs have no claim)
  - `doc_type: Mapped[DocumentType]` Enum column
  - `title: Mapped[str]` String(200), auto-generated default like "Demand Letter — 2026-04-05"
  - `input_data: Mapped[Optional[dict]]` using same `_JSONB_OR_JSON` pattern from claim.py (MutableDict + JSONB with JSON variant for SQLite)
  - `created_at`, `updated_at` DateTime(timezone=True) same pattern as Claim
  - Relationships: `user`, `claim` (back_populates), `versions` (one-to-many to DocumentVersion, order_by version_number desc)
  - Composite index on (user_id, doc_type)

- `DocumentVersion(db.Model)` on table `document_versions`:
  - `id: Mapped[int]` primary key
  - `document_id: Mapped[int]` FK to documents.id with CASCADE, indexed
  - `version_number: Mapped[int]` default 1
  - `input_data_snapshot: Mapped[Optional[dict]]` same JSONB pattern — snapshot of input_data at generation time
  - `generated_at: Mapped[datetime]` DateTime(timezone=True)
  - Relationship: `document` back to Document
  - Unique constraint on (document_id, version_number)

Add `documents` relationship to Claim model: add `documents: Mapped[list["Document"]] = relationship("Document", back_populates="claim")` in claim.py. Import Document in claim.py is NOT needed — use string reference.

**4. Create `app/documents/routes.py`:**

`from __future__ import annotations` at top.

Routes (all `@login_required`):

- `GET /documents/` — list all user's documents with latest version info. Query `Document.query.filter_by(user_id=current_user.id).order_by(Document.updated_at.desc()).all()`. Render `documents/index.html`.

- `GET /documents/new/<doc_type>` — start standalone document creation (no claim). Validate doc_type against DocumentType enum values. Create empty Document, redirect to review page.

- `GET /documents/from-claim/<int:claim_id>/<doc_type>` — start document from assessed claim. Verify claim ownership (`claim.user_id == current_user.id`) and status (`ClaimStatus.ASSESSED`), abort(403/404) otherwise. Create Document with claim_id set, pre-populate input_data from claim.step_data using a `_prepopulate_from_claim(claim, doc_type)` helper. Redirect to review page.

- `GET /documents/<int:doc_id>/review` — show review/supplement page. Verify ownership. Pre-populated fields from input_data plus empty fields for address_for_service, representative_name, representative_type (lawyer/paralegal/agent/self), court_location. Render `documents/review.html`. This is ONE page with all fields, not a multi-step wizard.

- `POST /documents/<int:doc_id>/review` — save review form data. Merge all form fields into document.input_data dict. Redirect to preview page.

- `GET /documents/<int:doc_id>/preview` — HTML preview of the final document. Verify ownership. Call `render_document_html(document)` from document_service. Render `documents/preview.html` which embeds the HTML output inline (not in an iframe).

- `GET /documents/<int:doc_id>/download` — generate PDF and return as download. Verify ownership. Create new DocumentVersion (increment version_number), snapshot input_data. Call `render_document_pdf(document)` from document_service. Return PDF response with `Content-Disposition: attachment; filename=...`.

- `GET /documents/<int:doc_id>/versions` — list all versions of a document. Verify ownership. Render `documents/versions.html`.

- `GET /documents/<int:doc_id>/versions/<int:version_id>/download` — re-generate PDF from a specific version's input_data_snapshot. Verify ownership. Return PDF.

- `GET /fees` — standalone filing fee page. No login required. Read all fee constants from ontario_constants.py. Render `documents/fees.html`.

Helper `_prepopulate_from_claim(claim, doc_type)` extracts from claim.step_data:
- For DEMAND_LETTER: claim_type, facts_description, incident_date, amount, opposing_party_name, opposing_party_type
- For FORM_7A: same as demand letter plus evidence data
- For FORM_9A: claim_type, opposing_party info (plaintiff becomes the other party)

**5. Create `app/services/document_service.py`:**

`from __future__ import annotations` at top.

- `render_document_html(document) -> str`: Dispatches to the correct Jinja2 template based on document.doc_type. Builds context dict from document.input_data. Returns rendered HTML string. For now, only DEMAND_LETTER is implemented; FORM_7A and FORM_9A raise NotImplementedError (plan 03-02 adds them).

- `render_document_pdf(document, input_data_override=None) -> bytes`: Calls render_document_html (or uses input_data_override for version re-generation), then converts to PDF via WeasyHTML(string=html).write_pdf(). Same pattern as existing pdf_service.py.

- `render_version_pdf(version) -> bytes`: Convenience wrapper that creates a temporary document-like object from version.input_data_snapshot and calls render_document_pdf.
