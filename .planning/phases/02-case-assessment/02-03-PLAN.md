1. **Create `app/services/assessment_service.py`** following research Pattern 4:
   - `from __future__ import annotations` as FIRST import
   - Define `_SYSTEM_PROMPT` constant — the system prompt for Claude that enforces statistical framing:
     ```
     You are a legal information assistant describing statistical patterns in Ontario Small Claims Court cases.

     Rules you MUST follow:
     - ONLY describe statistical patterns. Never give legal advice.
     - NEVER say "you should", "you must", "I recommend", "you will win", "you will lose".
     - Frame ALL observations as: "Cases with similar characteristics in Ontario..."
     - Do not predict the outcome of this specific case.
     - Keep response under 150 words.
     - End with: "This is a statistical observation, not legal advice."
     ```
   - Define `_MODEL = "claude-sonnet-4-6"` (pinned model alias)
   - Implement `build_claim_summary(claim) -> str`:
     - Extract all step_data fields into a structured text summary
     - Include: claim type, description of facts, amount, opposing party type, limitation status, evidence score
     - Do NOT include personally identifying information (names, addresses, emails) in the prompt sent to Claude
     - Return the summary string
   - Implement `get_case_strength_assessment(claim) -> GuardrailResult`:
     - Check `os.environ.get("ANTHROPIC_API_KEY")` — if absent, return a GuardrailResult with status=BLOCKED and text="AI assessment is not currently available. Complete your assessment without it, or try again later."
     - Also check `current_app.config.get("AI_ASSESSMENT_ENABLED", True)` — if False, return same blocked result (feature flag for lawyer review gate)
     - Build claim summary via `build_claim_summary(claim)`
     - Create Anthropic client, call `client.messages.create()` with _SYSTEM_PROMPT and claim summary
     - Pass raw response through `guardrail.process()` from ai_guardrail
     - Store the processed text in `claim.ai_assessment` and commit
     - Return the GuardrailResult
     - Wrap the entire Anthropic call in try/except — on any API error, return BLOCKED result with "AI assessment temporarily unavailable" message
   - Mark the entire module LAWYER_REVIEW_REQUIRED in a module docstring

2. **Add AI_ASSESSMENT_ENABLED config to `app/config.py`:**
   - `DefaultConfig`: `AI_ASSESSMENT_ENABLED = os.environ.get("AI_ASSESSMENT_ENABLED", "true").lower() == "true"`
   - This allows disabling AI assessment via env var without code changes (lawyer review gate)

     - Import `date` from datetime for the report date
     - Build a context dict with all claim data needed for the PDF:
       - Claim type label (map value to label from VALID_CLAIM_TYPES)
       - Facts summary (description, dates)
       - Amount and jurisdiction status
       - Limitation period result (status, deadline, days remaining)
       - Evidence inventory (items checked, score)
       - AI assessment text (if available, from claim.ai_assessment)
       - Report generation date
     - Call `render_template("assessment/pdf/assessment_report.html", **context)`
     - Call `WeasyHTML(string=html_string).write_pdf()`
     - Return pdf_bytes

     ```html

     </div>
     ```
   - Report sections (in order):
     a) Header: "Case Assessment Summary" + Bryan and Bryan branding + date
     b) Claim Type: display label and any complexity warnings
     c) Facts Summary: description, incident date, discovery date
