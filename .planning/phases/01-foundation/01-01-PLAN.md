    **app/services/auth_tokens.py** — Stateless magic link tokens:
    - MAGIC_LINK_SALT = "magic-link-auth"
    - MAGIC_LINK_MAX_AGE = 900 (15 minutes)
    - generate_magic_token(email: str) -> str: uses URLSafeTimedSerializer with current_app.config['SECRET_KEY'] and salt
    - verify_magic_token(token: str) -> str | None: loads token with max_age, returns email on success, None on SignatureExpired or BadSignature

    **app/auth/__init__.py** — Blueprint: auth_bp = Blueprint('auth', __name__)

    **app/auth/routes.py** — Three routes with rate limiting:

    GET+POST /login:
    - GET: render login.html with email input form
    - POST: rate limited (5/hour per IP via get_remote_address, 3/hour per email via custom key_func that reads request.form.get('email', 'anonymous').lower())
    - Validate email is present and looks like an email (basic check, not regex monster)
    - Generate magic token, build verification URL: url_for('auth.verify', token=token, _external=True)
    - If SENDGRID_API_KEY is configured, send email via sendgrid. If not configured (local dev), print the magic link to console/flash it (so dev works without SendGrid).
    - Always show "check your email" page regardless of whether email exists (prevent enumeration)
    - Redirect to check_email.html

    GET /verify/<token>:
    - Call verify_magic_token(token)
    - If None (expired/invalid): render link_expired.html with "request a new link" button
    - If valid email: look up User by email. If not found, create new User (create-on-verify pattern). Update last_login_at. Call login_user(user, remember=True) — MUST pass remember=True for 7-day sessions.
    - Redirect to next page: read `next` from query param, validate with url_has_allowed_host_and_scheme(next_page, request.host), default to url_for('main.index')

    GET /logout:
    - Call logout_user()
    - Flash "You have been logged out"
    - Redirect to auth.login
