## Remaining Tech Debt

### Phase 1: Foundation
- Docker build not runtime-tested — structural verification only. First push to main with GCP secrets configured will verify.

### Phase 2: Case Assessment
- **Hardcoded fee strings in `step_amount.html`** — `$249.00`/`$108.00` and `25000` threshold are literal strings rather than template expressions from `ontario_constants.py`. Will silently drift if fees change. Fix: pass constants to template context and replace literals.

### Phase 3: Documents and Guide
- **Hardcoded `max="50000"` in `review.html`** — Amount input cap is a literal attribute, not passed from route context. Will become wrong if monetary limit changes. Fix: pass `SMALL_CLAIMS_MONETARY_LIMIT` from `review()` route.
- Redundant `[x-cloak]` inline CSS in `guide/index.html` and `review.html` — harmless duplication of the rule in `main.css`.

### Phase 4: Dashboard and Deployment
- **REGULATORY_SIGNOFF.md unsigned** — Expected pre-launch gate. Supervising lawyer must review before external launch.
- **`User.claims` relationship uses `lazy="dynamic"`** — Deprecated in SQLAlchemy 2.x. Dashboard queries claims separately so no current break, but any direct iteration of `current_user.claims` would fail. Latent issue.

### Total: 6 items across 4 phases (0 blockers)
