## Gap Detail

### GAP-1: Form 7A/9A Creation Buttons Missing from Documents UI

**Severity:** Integration gap — backend fully supports Form 7A/9A but UI doesn't expose creation buttons

**What's broken:** The process guide (`/guide`) links users to `/documents/` with text promising "Generate your Form 7A using our tool" and "Generate your Form 9A using our tool." But `documents/index.html` only has buttons for Demand Letter creation. The routes (`documents.new_document` and `documents.from_claim`) accept `form_7a` and `form_9a` as valid `doc_type` values. The review template, document service, and PDF templates all handle all three types. The gap is solely missing `<a>` elements in the template.

**Impact:** A user following the guide's instructions for Form 7A or Form 9A will arrive at a page with no way to create those documents.

**Fix scope:** Add 4 buttons to `app/templates/documents/index.html` (2 for Form 7A, 2 for Form 9A — both from-claim and standalone paths, matching existing Demand Letter pattern).
